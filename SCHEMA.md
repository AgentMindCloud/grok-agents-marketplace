# featured-agents.json schema

The registry at [`featured-agents.json`](featured-agents.json) is the single
source of truth for the gallery. The CI job
[`validate_registry.py`](scripts/validate_registry.py) enforces this schema on
every PR; entries that do not conform are rejected.

This document describes the current working schema (**version `1.0`**). Fields
are stable — additions are additive, renames require a version bump.

## Top-level

```json
{
  "version": "1.0",
  "updated": "YYYY-MM-DD",
  "agents": [ /* AgentEntry, ... */ ]
}
```

| Field     | Type     | Required | Notes                                    |
|-----------|----------|----------|------------------------------------------|
| `version` | string   | yes      | Must equal `"1.0"`.                       |
| `updated` | string   | yes      | ISO-8601 date of the last registry edit. |
| `agents`  | array    | yes      | List of `AgentEntry` objects.            |

## `AgentEntry`

```json
{
  "name": "hello-grok",
  "description": "The simplest possible Grok agent. ...",
  "path": "templates/hello-grok",
  "tags": ["beginner-friendly", "tutorial"],
  "install_command": "grok-install install github.com/agentmindcloud/awesome-grok-agents/templates/hello-grok",
  "safety_profile": "standard",
  "certified": true,
  "poster": "docs/posters/hello-grok.svg"
}
```

| Field             | Type       | Required | Notes |
|-------------------|------------|----------|-------|
| `name`            | string     | yes      | Matches the template directory name. Must be unique in the registry. |
| `description`     | string     | yes      | Single sentence shown in the gallery and poster card. |
| `path`            | string     | yes      | Path from repo root to the template directory. Must exist on disk. |
| `tags`            | string[]   | yes      | Free-form tags for search/filtering. Keep lowercase, hyphenated. |
| `install_command` | string     | yes      | The exact `grok-install install …` one-liner for this template. |
| `safety_profile`  | enum       | yes      | One of `strict`, `standard`, `permissive`. Must match the template's `.grok/grok-security.yaml`. |
| `certified`       | boolean    | yes      | `true` only if the template meets the Grok-Native Certified bar (see [README](README.md#grok-native-certified)). |
| `poster`          | string     | yes      | Path to the SVG poster. File must exist; generated by `scripts/gen_poster.py`. |

## Invariants enforced by CI

- Every `name` is unique.
- Every `path` resolves to a directory that contains `grok-install.yaml`,
  `README.md`, `.env.example`, and a `.grok/` folder.
- Every `poster` path resolves to an existing file.
- No orphan templates: every directory under `templates/` must have a matching
  registry entry.
- No ghost entries: every registry entry must have a matching directory.
- `safety_profile` is one of the three allowed values.

## Adding a new entry

1. Append an `AgentEntry` to `agents`.
2. Bump `updated` to today's date.
3. Regenerate the poster: `python scripts/gen_poster.py <name>`.
4. Run `python scripts/validate_registry.py` — it must print
   `registry OK: N agents`.

Do **not** bump `version` for additions; only structural changes to the schema
require a version bump.
