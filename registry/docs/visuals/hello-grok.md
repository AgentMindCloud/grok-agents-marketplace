# hello-grok — visual design notes

`hello-grok` is the front door of the gallery: the first template a new
contributor will clone, read, and run. Its v2.14 visuals are deliberately
quiet.

## Design choice: minimal

```yaml
visuals:
  accent_color: "#7c5cff"
  preview_card:
    style: minimal
    title: hello-grok
    subtitle: Your first Grok agent in 20 lines of YAML.
    poster: docs/posters/hello-grok.svg
```

- **`style: minimal`** — no gradients, no glow, no animation. The preview
  card is a flat panel with a single accent bar. Contributors reading the
  template file shouldn't have to skim past a block of styling config to
  find the actual agent.
- **`accent_color: #7c5cff`** — the canonical Grok violet from
  `scripts/gen_poster.py`. Acts as the default when no per-template accent
  is specified, so `hello-grok` anchors the palette for everything else.
- **No `auto_generate.demo_media`** — beginners should see the template
  exactly as they'll ship it. A generated demo GIF would add magic that
  isn't in the YAML.
- **Poster reused** — `docs/posters/hello-grok.svg` is the canonical card
  already generated by `scripts/gen_poster.py`. No override needed.

## Why this matters

Every other template in the gallery can riff on this minimal baseline.
If `hello-grok` shipped with full futuristic visuals, the implicit
message would be "add visuals or you're not trying." Keeping it plain
sends the opposite — correct — message: visuals are optional, and the
agent itself is the product.
